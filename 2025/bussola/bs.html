Como é hoje

Objetivo

  • Permitir trocar a implementação de UI sem alterar o projeto
  • Usar tanto Vuetify quanto abstração customizada
  • Reduzir acoplamento e facilitar manutenção

Adapter Pattern

  • Define uma interface genérica
  • Cria adapters que traduzem chamadas para Vuetify ou abstração
  • Projeto só interage com a interface

Adapter Pattern: Exemplo


// Interface genérica
interface IButton {
  render(): void;
}

// Adapter Vuetify
class VuetifyButton implements IButton {
  render() {
    return `Clique`;
  }
}

// Adapter Abstração
class CustomButton implements IButton {
  render() {
    return `Clique`;
  }
}

// Uso
function renderButton(button: IButton) {
  document.body.innerHTML += button.render();
}

renderButton(new VuetifyButton());
renderButton(new CustomButton());

Bridge Pattern

  • Separa abstração da implementação concreta
  • Permite trocar implementação dinamicamente
  • Boa para projetos que mudam a UI em runtime

Bridge Pattern: Exemplo


// Implementações concretas
class VuetifyImpl {
  renderButton() { return `Clique`; }
}

class CustomImpl {
  renderButton() { return `Clique`; }
}

// Bridge
interface Button {
  render(): string;
}

class ButtonBridge implements Button {
  constructor(private impl: { renderButton: () => string }) {}
  render() {
    return this.impl.renderButton();
  }
}

// Uso
const vuetifyButton = new ButtonBridge(new VuetifyImpl());
const customButton = new ButtonBridge(new CustomImpl());

document.body.innerHTML += vuetifyButton.render();
document.body.innerHTML += customButton.render();

Bridge Pattern: de forma Sucinta


import BsButton from '@/components/_bs/BsButton.vue'
import BsForm from '@/components/_bs/BsForm.vue'
import BsTextField from '@/components/_bs/BsTextField.vue'

app.component('BsButton', BsButton)

Conclusão

  • Adapter: bom para compatibilizar APIs diferentes
  • Bridge: bom para trocar implementação dinamicamente
  • Ambos ajudam a desacoplar UI do projeto
