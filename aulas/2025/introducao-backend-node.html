O que é Backend?

  • Responsável por regras de negócio, acesso a dados e segurança
  • Executado no servidor
  • Diferente do frontend (navegador)

Comunicação Cliente x Servidor

  • Requisições HTTP (GET, POST, PUT, DELETE)
  • Cliente envia requisição → servidor responde
  • Baseado em protocolos

O que é JSON?

  • JavaScript Object Notation
  • Formato leve de troca de dados
  • Base para comunicação entre front e back
{
  "nome": "Chiquim",
  "idade": 42
}

O que é uma API?

  • Application Programming Interface
  • Permite que sistemas diferentes se comuniquem
  • Backends modernos expõem APIs REST ou GraphQL

Banco de Dados (Breve Introdução)

  • Estrutura para armazenar dados
  • Relacionais (MySQL, PostgreSQL)
  • Não relacionais (MongoDB, Redis)
  • Conectado ao backend por bibliotecas/ORMs

Comparativo: Node.js vs outros backends

  • PHP (Laravel)
  • Java (Spring Boot)
  • Python (Django, Flask)
  • Ruby (Rails)
  • Node.js: rápido, não bloqueante, JS no back

Por que Node.js?

  • Mesma linguagem no front e no back
  • Rápido e leve
  • Grande ecossistema (npm)
  • Ideal para aplicações modernas e APIs

Mão na Massa: Node.js

  • Criar projeto com npm init
  • Instalar express
  • Criar primeira rota: GET /
  • Testar com Postman ou curl
