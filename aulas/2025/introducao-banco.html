Introdução a Banco de Dados

Conceitos básicos e primeiros passos

Por que precisamos de um Banco de Dados?

  • Armazenar informações de forma organizada
  • Facilidade de acesso e busca
  • Segurança e integridade dos dados
  • Compartilhamento de informações

Profissões

  • DBA - DataBase Administrator
  • Data Analytics
  • Cientista de Dados
  • Devs Backend

O que é um Banco de Dados?

Conjunto organizado de dados, geralmente armazenados e acessados eletronicamente a partir de um sistema de computador.

Conceitos Fundamentais

  • Tabelas (Tables)
  • Colunas (Columns)
  • Registros (Rows)
  • Índices (Indexes)
  • Relacionamentos (Relationships)

Exemplo de Tabela

+----+----------+-----------+
| ID | Nome     | Email     |
+----+----------+-----------+
|  1 | João     | joao@x.com|
|  2 | Maria    | maria@x.com|
+----+----------+-----------+

Principais Bancos de Dados

  • MySQL
  • PostgreSQL
  • SQL Server
  • Oracle
  • MongoDB (NoSQL)
  • Redis (NoSQL)

Banco Relacional vs NoSQL

  • Relacional: Estrutura em tabelas, SQL (Ex: MySQL, PostgreSQL)
  • NoSQL: Estrutura flexível (documentos, chave-valor, etc) (Ex: MongoDB, Redis)

Principais Comandos SQL

  • DDL - Data Definition Language (CREATE, ALTER, DROP)
  • DML - Data Manipulation Language (INSERT, UPDATE, DELETE)
  • DQL - Data Query Language (SELECT)
  • DCL - Data Control Language (GRANT, REVOKE)

DDL - Comandos de Estrutura

CREATE TABLE usuarios (id INT, nome VARCHAR(100));
ALTER TABLE usuarios ADD email VARCHAR(100);
DROP TABLE usuarios;

DML & DQL - Manipulando e Consultando Dados

INSERT INTO usuarios (id, nome) VALUES (1, 'João');
UPDATE usuarios SET nome = 'Maria' WHERE id = 1;
DELETE FROM usuarios WHERE id = 1;
SELECT * FROM usuarios;
