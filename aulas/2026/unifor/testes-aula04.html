Integrando a qualidade à esteira de entrega contínua

Automação vs. Teste Contínuo

  • Teste Automatizado: Escrever um script que executa um teste sem intervenção humana. (Foco na técnica)
  • Teste Contínuo: Executar esses testes de forma ininterrupta como parte do pipeline de liberação. (Foco no processo)
  • Objetivo principal: Fornecer feedback instantâneo sobre os riscos de negócio de um commit.

A Anatomia de um Pipeline de CI

Cada alteração de código dispara um ciclo automático:

  1. Build: Compilação e empacotamento.
  2. Testes Unitários: Feedback em segundos.
  3. Análise Estática: Qualidade de código e segurança (Lint/SonarQube).
  4. Testes de Integração/API: Validação de contratos.
  5. Deployment (Staging): Liberação em ambiente controlado.

Quality Gates: Os Guardiões

Um portão de qualidade define as regras para o código avançar para produção:

  • ❌ Falha em qualquer teste automatizado.
  • ❌ Cobertura de código inferior a 85%.
  • ❌ Presença de credenciais ou senhas expostas (Hardcoded Secrets).
  • ❌ Vulnerabilidades críticas de dependências (CVEs).

Estratégia de Otimização: Paralelismo

Pipelines lentos matam a produtividade do time de engenharia.

  • Se sua suíte de testes demora 1 hora para rodar, os devs vão odiar o CI.
  • Solução DevOps: Dividir os testes para rodarem em paralelo em múltiplos containers/máquinas (Múltiplos Runners).

Criando um Pipeline no GitHub Actions

Vamos configurar o ambiente para que o GitHub rode o comando npm test a cada Pull Request.

Estrutura de Pastas do Projeto

Para o GitHub Actions funcionar, precisamos criar uma pasta oculta na raiz do projeto:


meu-projeto/
├── .github/
│   └── workflows/
│       └── ci.yml   <-- O pipeline vive aqui!
├── app.js
├── app.test.js
└── package.json

O Arquivo de Configuração (ci.yml)


name: Pipeline de Testes Continuos

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  testar-aplicacao:
    runs-on: ubuntu-latest

    steps:
    - name: Baixar codigo do repositorio
      uses: actions/checkout@v3

    - name: Configurar o Node.js
      uses: actions/setup-node@v3
      with:
        node-node: 18

    - name: Instalar Dependencias
      run: npm install

    - name: Executar os Testes Automatizados
      run: npm test -- --coverage

Desafio de Hoje 🚀

  1. Criar a pasta .github/workflows/ no seu repositório do trabalho.
  2. Adicionar o arquivo ci.yml.
  3. Subir um commit propositalmente quebrando um teste e ver o GitHub Actions bloquear o avanço.
  4. Consertar o teste e ver a esteira ficar verde (Sucesso).
