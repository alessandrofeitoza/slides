Padrões GRASP

Atribuição de Responsabilidades com Qualidade

Disciplina: ARA-0110: Padrões de Projeto

O que é GRASP?

General Responsibility Assignment Software Patterns

  • Não são "receitas", são princípios.
  • Foco em decidir: Qual objeto faz o quê?
  • Complementares os padrões GoF (Design Patterns).

Termos principais

  • Especialista na Informação
  • Criador/Creator
  • Controlador
  • Baixo Acoplamento
  • Alta Coesão
  • Polimorfismo
  • Fabricação Pura
  • Indireção
  • Variações Protegidas

1. Especialista na Informação

Atribua a responsabilidade à classe que possui a informação necessária para realizá-la.

Exemplo: Cálculo de Total

Java 

public class Venda {
    private List<Item> itens;

    // A Venda é a especialista, pois conhece seus itens
    public double getTotal() {
        double total = 0;
        for(Item i : itens) {
            total += i.getSubtotal(); 
        }
        return total;
    }
}
                    

public class Livro {
    private String titulo;
    private int totalExemplares;
    private int exemplaresEmprestados;

    public Livro(String titulo, int total) {
        this.titulo = titulo;
        this.totalExemplares = total;
    }

    public boolean podeSerEmprestado() {

    }

    public void registrarEmprestimo() {
        if (podeSerEmprestado()) {

        }
    }

2. Criador (Creator)

Quem deve criar o objeto A? A classe B, se:

  • B agrega ou contém A.
  • B utiliza A de forma muito próxima.
  • B possui os dados de inicialização de A.

public class Pedido {
    private List<LinhaItem> itens;

    // Pedido é o Criador natural de LinhaItem
    public void adicionarItem(Produto p, int qtd) {
        itens.add(new LinhaItem(p, qtd));
    }
}

4. Baixo Acoplamento

Minimize as dependências entre classes. Facilita a manutenção e reuso.

5. Alta Coesão

Mantenha as responsabilidades de uma classe focadas e relacionadas. Evite "Classes Deus".

6. Polimorfismo

Trate variações de comportamento baseadas no tipo sem usar switch/case ou if/else excessivos.


public interface FormaPagamento {
    void pagar(double valor);
}

public class Cartao implements FormaPagamento {
    public void pagar(double v) { /* lógica cartão */ }
}

public class Pix implements FormaPagamento {
    public void pagar(double v) { /* lógica pix */ }
}

8. Indireção

Atribui a responsabilidade a um objeto intermediário para mediar entre outros componentes.

Muito comum em implementações de Adaptadores.

9. Variações Protegidas

Proteja os elementos internos de variações de outros elementos através de interfaces.

"Encapsule o que varia."
