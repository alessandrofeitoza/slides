Composite • Observer • Strategy

1. Composite Pattern

Cenário: Árvores de Componentes (DOM/React)

O Problema Real 🧩

Como criar um sistema de permissões em um Menu? Alguns itens são Links simples, outros são Dropdowns que contêm mais Links.

Se eu esconder o "Pai", todos os "Filhos" devem sumir automaticamente.

A Solução Web

Tratamos um <ButtonGroup /> (Composite) exatamente como um <Button /> (Leaf). Ambos respondem ao método render().

Onde você usa isso?

  • DOM do Navegador: Uma div pode conter outras divs ou apenas texto.
  • Flutter/React: Column e Row são Composites de Widgets.

2. Observer Pattern

Cenário: Reatividade e Eventos

O Problema Real ⚡

Você tem um carrinho de compras. Quando o usuário clica em "Adicionar", três lugares precisam atualizar:

  • O ícone do topo (contador).
  • A listagem lateral (modal).
  • O cálculo do frete.

A Solução Web

O Store (Subject) notifica todos os Componentes (Observers) que o estado mudou.


// Exemplo simplificado de Redux/Vuex
store.subscribe(() => {
  console.log("O estado mudou! Atualizando UI...");
});

Exemplos no Dia a Dia

  • React `useEffect`: Observando mudanças em dependências.
  • WebSockets: O servidor avisa o app (cliente) que chegou mensagem.
  • Firebase: `onSnapshot` para atualizações em tempo real.

3. Strategy Pattern

Cenário: Lógica de Negócio Intercambiável

O Problema Real 💳

Seu app aceita Stripe, PayPal e Pix. Cada um tem uma API diferente.

O Erro: Encher seu componente de Checkout com if(tipo == 'stripe').

A Solução Web

Defina uma interface PaymentProcessor. O código do seu App não sabe quem está processando, apenas que o método pay() existe.

Aplicações Reais

  • Auth: Login via Google, Apple ou E-mail (Estratégias de Passport.js).
  • Upload de Arquivos: Enviar para S3, Firebase Storage ou Local.
  • Validação de Forms: Aplicar regras diferentes (Regex, Tamanho, Email) dinamicamente.
