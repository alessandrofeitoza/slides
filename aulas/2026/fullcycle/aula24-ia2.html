IA para Análise de Falhas e Performance

SRE, Observabilidade e Diagnóstico Assistido

Nem toda IA mas sempre uma IA

O Problema Real

No mercado, falhas não avisam. O custo do minuto parado é medido em milhares de reais.

Gargalos de Infra

  • CPU Steal (Cloud)
  • I/O Wait (DB)
  • OOM Killer (K8s)

Gargalos de App

  • Deadlocks SQL
  • Race Conditions
  • Event Loop Blocking

Métricas que a IA ajuda a baixar

MTTR

Mean Time To Resolution (Tempo Médio de Solução)

MTTD

Mean Time To Detection (Tempo Médio de Detecção)

"A IA não resolve o problema, ela reduz o tempo de hipótese."

A IA como Copiloto SRE

Como transformar um Log ruidoso em uma solução acionável.

Exemplo: Exception Laravel / PHP


[2024-05-14 18:22:10] production.ERROR:
SQLSTATE[HY000]: General error: 1205 Lock wait timeout exceeded;
try restarting transaction (Connection: mysql, SQL: update `orders`
set `status` = 'processed' where `id` = 99283)

Hipótese Gerada por IA: Transação longa aberta em outro Worker está segurando o Row Lock do ID 99283. Verifique o PaymentGatewayJob.

Exemplo: Memory Leak Node.js


<--- Last few GCs --->
[1422:0x559e210] 30221 ms: Mark-sweep 2042.1 (2050.2) -> 2038.5 (2050.7) MB
[1422:0x559e210] 31250 ms: Mark-sweep 2040.5 (2050.7) -> 2039.2 (2051.1) MB
FATAL ERROR: Ineffective mark-compacts near heap limit Allocation failed

IA identifica imediatamente: Heap Exhaustion. Possível closure acumulando objetos no escopo global.

Engenharia de Prompt Técnico

Pare de perguntar como um iniciante. Pergunte como um Arquiteto.

O Framework de Prompt SRE

  1. Contexto: Stack (Node v20, K8s, Redis).
  2. Input: Log bruto ou Stack Trace.
  3. Sintomas: "API lenta apenas em rotas de escrita".
  4. Constraints: "Não sugira aumentar recursos de hardware".

Prompt de Ouro (Copie e Use)


Atue como um Engenheiro de Performance. Analise o log abaixo.
1. Identifique a Causa Raiz Provável.
2. Explique o impacto no Event Loop/Threads.
3. Sugira uma correção imediata e uma estrutural.

Log: {{COLAR_LOG_AQUI}}

Deep Dive: IA e Performance

O Inimigo Silencioso:


// Código Ruim
$posts = Post::all();
foreach($posts as $post) {
    echo $post->author->name;

N+1 Queries

Análise IA

"Detectei 100 queries SQL para 100 posts. Use Eager Loading."


$posts = Post::with('author')->get();

Gargalos de Infra com IA

  • Análise de Thread Dumps.
  • Interpretação de gráficos de Flame Graphs.
  • Sugestão de tuning de sysctl e nginx.conf.

Limitações e Riscos

Alucinações

IA pode inventar flags de configuração que não existem no PostgreSQL.

Segurança

Nunca envie PII (Dados Pessoais) ou chaves de API nos logs para a IA.

"Confie, mas valide com a documentação oficial."

Desafio Final: War Room

Cenário: O banco de dados MySQL subiu para 100% de CPU. O Redis está respondendo com 2s de latência.

  1. Cole os logs no modelo de IA.
  2. Identifique se o problema é o Slow Query ou Connection Pool.
  3. Apresente a solução em 10 minutos.
