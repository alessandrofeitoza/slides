Nível 3: JS: O que há de errado?


                function calc(a) {
                  let r = 0;
                  for (let i = 0; i < a.length; i++) {
                    r += a[i];
                  }
                  return r;
                }
Problema: Nomes de variáveis genéricos e falta de métodos funcionais (Clean Code).

Nível 3: JS Refatorado!


                function sumArrayElements(numbers) {
                  return numbers.reduce((acc, curr) => acc + curr, 0);
                }

Melhoria: Nomes semânticos e uso de reduce.

Nível 4: JS O Desafio do If-Else


                function getPrice(type) {
                  if (type === 'apple') {
                    return 1.5;
                  } else if (type === 'banana') {
                    return 1.0;
                  } else if (type === 'cherry') {
                    return 2.5;
                  } else {
                    return 0;
                  }
                }

Nível 4: JS Object Literal


                function getPrice(type) {
                  const prices = {
                    apple: 1.5,
                    banana: 1.0,
                    cherry: 2.5
                  };
                  return prices[type] || 0;
                }

Nível 5: Banco de Dados

O que há de perigoso e ineficiente aqui?


    -- Buscando dados do usuário logado
    SELECT * FROM usuarios
    WHERE id = " + id_do_input + "
    AND ativo = 1;

Nível 5: Projeção exata


    -- Proteção contra SQL Injection e Performance
    SELECT nome, email, avatar FROM usuarios
    WHERE id = " + id_do_input + "
    AND ativo = 1
    LIMIT 1;

Melhoria: seleção de campos específicos e LIMIT 1.

Nível 5: Prepared Statements


    -- Proteção contra SQL Injection e Performance
    SELECT nome, email, avatar FROM usuarios
    WHERE id = ?
    AND ativo = 1
    LIMIT 1;

Melhoria: Uso de placeholders (?)

Nível 7: Transação de Saldo

O que acontece se o servidor cair na linha 3?


    -- Retira 100 da conta A
    UPDATE contas SET saldo = saldo - 100 WHERE id = 1;

    -- Adiciona 100 na conta B
    UPDATE contas SET saldo = saldo + 100 WHERE id = 2;

Nível 7: Atomicidade (ACID)


    START TRANSACTION;

    UPDATE contas SET saldo = saldo - 100 WHERE id = 1;
    UPDATE contas SET saldo = saldo + 100 WHERE id = 2;

    COMMIT;

Melhoria: Uso de Transações. Ou tudo acontece, ou nada acontece (Rollback automático em caso de erro).

PHP

Nível 8: PHP "das antigas"


    function get_user_name($data) {
        if (isset($data['name'])) {
            return $data['name'];
        } else {
            return 'Visitante';
        }
    }

Nível 8: PHP Moderno


    function getUserName(array $data): string {
        return $data['name'] ?? 'Visitante';
    }

Melhoria: Type hinting (array/string) e operador ??.

Nível 9: PHP e Objetos

O que acontece se o usuário não for encontrado?


    function getProfile($id) {
        $user = User::find($id);
        return $user->name;
    }

Nível 9: Nullsafe & Types


    function getProfile(int $id): ?string {
        return User::find($id)?->name ?? 'Usuário não encontrado';
    }

Melhoria: ?string (permite null), operador nullsafe ?-> e valor padrão.

TESTES

Nível 13: Testes de Unidade

O que falta para este teste ser confiável?


    test('deve somar dois números', () => {
      const resultado = somar(2, 2);
      expect(resultado).toBe(4);
    });

Nível 13: Edge Cases


    test('deve somar dois números e tratar entradas inválidas', () => {
      expect(somar(2, 2)).toBe(4);

      // Testando os limites (Edge Cases)
      expect(somar("2", 2)).toThrow("Input deve ser numérico");
      expect(somar(null, 2)).toThrow();
      expect(somar(0.1, 0.2)).toBeCloseTo(0.3);
    });

Melhoria: Testar tipos errados, valores nulos e precisão de ponto flutuante.

Nível 14: Teste Frágil

Por que este teste unitário vai falhar no CI (integração contínua)?


    test('deve buscar dados do usuário', async () => {
      // Faz uma chamada real para a API de produção
      const user = await UserService.fetchFromServer(1);

      expect(user.name).toBe('John Doe');
    });

Nível 14: Mocking (Dublês)


    test('deve buscar dados usando Mock', async () => {
      const spy = jest.spyOn(UserService, 'fetchFromServer')
        .mockResolvedValue({ id: 1, name: 'John Doe' });

      const user = await UserService.getById(1);

      expect(user.name).toBe('John Doe');
      expect(spy).toHaveBeenCalledWith(1);
    });

Melhoria: O teste não depende da API externa. É rápido, isolado e determinístico.
