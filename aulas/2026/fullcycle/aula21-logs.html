A "Caixa Preta"

Por que Logar?

  • Visibilidade: O que acontece em produção?
  • Auditoria: Histórico de ações críticas.
  • Monitoramento: Identificar erros proativamente.
  • Compliance: Requisitos legais (LGPD).

Logs e Compliance

Por que o jurídico se importa?

é o conjunto de regras e normas que uma empresa deve seguir. No mundo dos dados, os logs são a sua prova real.

  • Quem acessou o quê?
  • Quando um dado foi alterado?
  • Por que um usuário foi deletado?

LGPD e o "Rastro" de Dados

A Lei Geral de Proteção de Dados exige que empresas saibam o ciclo de vida dos dados pessoais.

  • Auditoria: Você precisa provar que o consentimento foi dado.
  • Incidentes: Em caso de vazamento, você deve reportar o que foi exposto.
  • Direito ao Esquecimento: O log deve registrar a exclusão, mas não pode manter o dado sensível.

Lembra da Renner (2021)?

Princípio da Responsabilização

"O agente deve demonstrar a adoção de medidas eficazes e capazes de comprovar a observância e o cumprimento das normas."

Padrões de Mercado

Sigla Onde se aplica Exigência de Log
PCI-DSS Pagamentos / Cartão Trilha de auditoria para todo acesso a dados de cartões.
HIPAA Saúde (Health Techs) Logs estritos de quem visualizou prontuários médicos.
SOC2 SaaS / Cloud Monitoramento de acessos para garantir segurança e disponibilidade.

O Dilema do Desenvolvedor

Se um dado sensível cai no log, o log se torna o vazamento.

Risco de Compliance

Logar o payload completo da requisição incluindo a senha ou o CPF do cliente.

Solução: Masking (Anonimizar)

Substituir dados sensíveis antes de gravar:
"cpf": "***.456.***-01"

E como "logar"?

Log no PHP "Raiz" (Sem Framework)

<?php
error_log("Usuário logado com sucesso!", 0);

function writeLog($level, $message) {
    $date = date('Y-m-d H:i:s');

    $logEntry = "[$date] [$level] $message" . PHP_EOL;

    file_put_contents(__DIR__ . '/app.log', $logEntry, FILE_APPEND);
}

<?php
try {
    throw new Exception("Falha na conexão com o Banco");
} catch (Exception $e) {
    writeLog(
        'ERROR',
        $e->getMessage(),
        "No arquivo ".__FILE__." na linha ".__LINE__
    );
}
?>

Backend vs Frontend

Backend

Foco em infra, lógica e banco. Logs ficam no servidor ou agregadores.

Frontend

Foco em UX e Erros de JS. Desafio: Capturar logs na máquina do cliente.

Qualidade do Log

✅ Bom: [ERROR] Payment fail. User #42. Reason: Timeout.

❌ Ruim: [ERROR] Algo deu errado.

"Se você loga tudo, você não tem nada. Se loga pouco, está cego."

Cuidado: O que NÃO logar

  • Senhas de usuários.
  • Dados Sensíveis (CPF, Cartão).
  • Logs em excesso (I/O excessivo).
  • Informações que violem a LGPD.

E AGORA?

Logs Estruturados vs. Texto

❌ Texto Simples (Unstructured)

[2023-10-27] User 42 bought item 100

Difícil de filtrar, requer Regex complexo para análise de dados.

✅ JSON Estruturado

{
  "timestamp": "2023-10-27",
  "user_id": 42,
  "action": "purchase",
  "item_id": 100
}

Busca instantânea e agregação nativa em ferramentas modernas.

Correlation ID (Traceability)

Como rastrear um erro em múltiplos serviços?

1. Frontend gera um ID único: req-789-abc

2. API Auth recebe o ID e loga com ele.

3. API Pedidos recebe o mesmo ID via Header.

Resultado: Você busca por req-789-abc e vê a jornada completa da requisição.

Os 3 Pilares da Observabilidade

Logs

Eventos imutáveis e detalhados. "O que aconteceu?"

Metrics

Dados numéricos agregados. "Quantas vezes/Qual a carga?"

Tracing

Caminho da requisição fim-a-fim. "Onde demorou?"

Na Prática: Contexto é tudo

❌ Ruim (Mudo):

try {
  saveOrder(order);
} catch (e) {
  console.log("Erro ao salvar"); // O que falhou? Qual pedido?
}

✅ Bom (Rastreável):

try {
  saveOrder(order);
} catch (error) {
  logger.error("Falha ao processar pedido", {
    order_id: order.id,
    user_id: user.id,
    reason: error.message,
    stack: error.stack
  });
}

🚀 Desafio Prático

  1. Configurar um Logger (Winston/Monolog).
  2. Criar um Middleware para gerar Correlation ID em cada request.
  3. Simular um erro e capturá-lo no Sentry ou Papertrail.
  4. Verificar se o log contém os metadados necessários para o debug.

Erros e Gargalos comuns

  • I/O de Disco: Escrita síncrona trava o app.
  • Falta de Rotação: Arquivos gigantes que lotam o HD.
  • Logs Silenciosos: Catch sem log.

Ecossistema PHP

  • Monolog: O padrão (Laravel/Symfony).
  • Sentry: Gestão de exceções.
  • Papertrail: Centralização em nuvem.

Ecossistema Node.js

  • Winston: Versátil e robusto.
  • Pino: Focado em alta performance.
  • Bunyan: Estrutura JSON nativa.

Dicas e Macetes

  1. Use JSON Estruturado.
  2. Implemente Correlation IDs (Rastreio).
  3. Centralize (ELK Stack ou Grafana Loki).
  4. Automatize alertas críticos.
